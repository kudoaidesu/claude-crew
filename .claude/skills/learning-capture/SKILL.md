# 学習記録スキル

## 目的

AIは同じ間違いを繰り返す。ユーザーの指摘から学んだ普遍的な原則をスキルとして記録し、将来のタスクで活用する。

## 発動条件

以下のいずれかに該当したとき：

1. ユーザーの指摘で「なるほど」と気づいた
2. 同じパターンが他のタスクでも使えそう
3. 「最初からこうすべきだった」と感じた
4. 繰り返し書いているコードやチェックがある

## ワークフロー

### 1. 気づきを言語化する

ユーザーに以下を確認する：

```
この気づきを普遍的な原則としてスキルにまとめてよろしいですか？

## 気づいたこと
[具体的な学び]

## なぜ最初から気づけなかったか
[AIの限界や盲点]

## 適用範囲
[どんなタスクで使えるか]
```

### 2. ユーザーの承認を得る

**必ず許可を取る。** 勝手にスキルを作成しない。

承認を求めるときの例：

> 「この原則をスキルとして記録してよいですか？既存のスキルに追記/新規スキル作成、どちらが適切でしょうか？」

### 3. skill-creatorで作成する

承認を得たら：

1. `skill-creator`スキルを参照
2. `init-skill.ts`でスキルを初期化
3. SKILL.mdを簡潔に書く（500行以下）
4. 必要なら`references/`に詳細を分離

### 4. 関連スキルから参照する

新しいスキルを作成したら、関連する既存スキルから参照を追加する。

例：`design-polish`を作成 → `frontend-design`から参照

## 良いスキルの条件

- **普遍的**: 特定のタスクだけでなく、広く適用可能
- **簡潔**: 核心を短く伝える（冗長な説明は不要）
- **具体的**: コード例やチェックリストで実践的
- **検索可能**: descriptionに発動条件を明記

## 悪いスキルの例

- プロジェクト固有すぎる（他で使えない）
- 当たり前すぎる（Claudeが元から知っている）
- 長すぎる（コンテキストを圧迫）
