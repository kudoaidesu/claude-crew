# アカデミックトレンド調査

## ワークフロー

### 1. 調査テーマの特定

ユーザーの質問から調査対象を明確化:
- **技術領域**: フロントエンド、DB設計、認証、検索、地図、etc.
- **具体的な関心**: アルゴリズム、アーキテクチャ、UXパターン、etc.
- **時期**: 直近1-2年のトレンドを中心に

### 2. 情報源の検索

優先度順に検索:

#### 学術論文
```
Web検索クエリ例:
- "キーワード site:arxiv.org 2025 OR 2026"
- "キーワード site:dl.acm.org recent"
- "キーワード survey paper 2025"
```

#### カンファレンス・発表
```
Web検索クエリ例:
- "キーワード conference 2025 2026"
- "Next.js conf 2025 キーワード"
- "React conf キーワード"
- "キーワード JSConf OR NodeConf 2025"
```

#### 技術ブログ・RFC
```
Web検索クエリ例:
- "キーワード site:vercel.com/blog"
- "キーワード site:supabase.com/blog"
- "キーワード RFC github"
- "キーワード engineering blog 2025"
```

#### ベンチマーク・比較調査
```
Web検索クエリ例:
- "キーワード benchmark comparison 2025"
- "State of JS 2025 キーワード"
- "Web Almanac キーワード"
```

### 3. トレンド分析

収集した情報を以下の軸で整理:

1. **主流のアプローチ**: 現在最も採用されている方法
2. **新興のアプローチ**: 注目を集めている新しい方法
3. **衰退するアプローチ**: 使われなくなりつつある方法
4. **当プロジェクトとの関連**: 直接適用可能か、参考になるか

## プラン作成時の必須ステップ

**プラン作成時（EnterPlanMode）は、必ずトレンド調査を含めること。**

実装計画を策定する際に、以下の手順でトレンド調査を組み込む:

1. 実装対象の技術領域を特定
2. 上記ワークフローに沿って最新トレンドを調査
3. プラン内の「調査結果」セクションに結果を記載
4. トレンドを踏まえた実装方針を提案

これにより、古い手法に固執せず、最新のベストプラクティスを反映した計画を立てられる。

## 注意事項

- 論文の内容を鵜呑みにせず、実用性・再現性を考慮
- 個人開発の制約（月額1万円以内）を踏まえた現実的な提案
- 最新≠最良。安定性・エコシステムの成熟度も考慮
